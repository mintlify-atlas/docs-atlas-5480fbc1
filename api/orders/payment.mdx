`POST /api/payment-order`

This endpoint creates an order record in the database and returns the payment data required to initiate a transaction via the [Bold](https://bold.co) payment gateway. The `totalPayment` is calculated server-side by applying any product discounts, so the client does not need to compute a total.

<Note>
  This is a custom route and does **not** follow the standard `POST /api/orders` path. The path is `/api/payment-order`.
</Note>

## How it works

1. The server receives the `products`, `user`, `currency`, and `address` from the request body.
2. For each product, the server applies any discount using: `discountedPrice = price - (price * discount / 100)`. Products with no discount use the original price.
3. `totalPayment` is computed as the sum of `discountedPrice * quantity` for all products, then rounded using `Math.round()`.
4. A UUID (`idPayment`) is generated as the order reference.
5. The order is saved with initial state `"Pedido creado"`.
6. A SHA-256 integrity hash is computed as: `SHA-256(idPayment + totalPayment + currency + SECRET_KEY_BOLD)`.
7. The response includes the order record plus `hashIntegraty` and `apiKey` for the Bold checkout widget.

## Request

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from the authentication endpoint.

  Format: `Bearer <token>`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`.
</ParamField>

### Body

<ParamField body="products" type="object[]" required>
  Array of product objects representing the items in the order. The server uses `price` and `discount` to compute the final total — do not pre-compute a total on the client.

  <Expandable title="Product item">
    <ParamField body="id" type="number" required>
      Product ID.
    </ParamField>
    <ParamField body="price" type="number" required>
      Unit price of the product before any discount, in the smallest currency unit (e.g., COP pesos).
    </ParamField>
    <ParamField body="discount" type="number" required>
      Discount percentage to apply (0–100). Pass `0` for no discount.
    </ParamField>
    <ParamField body="quantity" type="number" required>
      Number of units being ordered.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="user" type="number" required>
  The numeric ID of the authenticated user placing the order.
</ParamField>

<ParamField body="currency" type="string" required>
  ISO 4217 currency code used for the transaction. Used in the Bold integrity hash.

  Example: `"COP"`
</ParamField>

<ParamField body="address" type="object" required>
  Shipping address to associate with the order. This is stored as a snapshot and is not linked to a separate address record.

  <Expandable title="Address fields">
    <ParamField body="title" type="string" required>
      Address label shown to the customer (e.g., `"Casa"`, `"Oficina"`).
    </ParamField>
    <ParamField body="name" type="string" required>
      Recipient's full name.
    </ParamField>
    <ParamField body="address" type="string" required>
      Street address.
    </ParamField>
    <ParamField body="city" type="string" required>
      City.
    </ParamField>
    <ParamField body="state" type="string" required>
      State or department.
    </ParamField>
    <ParamField body="postalCode" type="string" required>
      Postal code.
    </ParamField>
    <ParamField body="phone" type="string" required>
      Phone number without country code.
    </ParamField>
    <ParamField body="dialCode" type="string" required>
      Country dial code, e.g. `"+57"`.
    </ParamField>
  </Expandable>
</ParamField>

## Response

The response is a flat object (not wrapped in `data`) containing the full order record plus the Bold payment fields.

<ResponseField name="id" type="number">
  Numeric ID of the newly created order.
</ResponseField>

<ResponseField name="documentId" type="string">
  Strapi v5 document identifier.
</ResponseField>

<ResponseField name="totalPayment" type="number">
  Server-computed order total after applying discounts and `Math.round()`. Use this value — not a client-side calculation — when configuring the Bold payment widget.
</ResponseField>

<ResponseField name="idPayment" type="string">
  UUID generated for this order. Used as the order reference (`orderId`) in the Bold integrity hash.
</ResponseField>

<ResponseField name="state" type="string">
  Always `"Pedido creado"` at creation. Updated by the payment webhook after Bold confirms the transaction.
</ResponseField>

<ResponseField name="products" type="object[]">
  The product snapshot passed in the request body, as stored in the order.
</ResponseField>

<ResponseField name="addressShiping" type="object">

    <ResponseField name="dialCode" type="string">Country dial code.</ResponseField>
  </Expandable>

## totalPayment calculation

The server computes `totalPayment` using the following logic for each product:

```
if discount > 0:
  effectivePrice = price - (price * discount / 100)
else:
  effectivePrice = price

total += effectivePrice * quantity

totalPayment = Math.round(total)
```

For the example request above:
- Cortado: `8500 * 2 = 17000`
- Filtrado (10% off): `12000 - 1200 = 10800 * 1 = 10800`
- Total: `Math.round(17000 + 10800) = 27800`

<Note>
  Always use the `totalPayment` value from the response when initializing the Bold widget. Discrepancies between client-computed totals and the server-computed `totalPayment` will cause the Bold integrity check to fail.
</Note>
