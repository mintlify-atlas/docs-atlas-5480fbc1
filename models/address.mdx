---
title: "Address"
description: "The Address data model stores a user's saved delivery addresses."
---

The `Address` model represents a named delivery address in a user's address book. Users can save multiple addresses (e.g., home, office) and select one at checkout. When an order is placed, the chosen address is copied as a JSON snapshot into the [Order](/models/order) record — the `Address` entry itself is never modified by the order process.

Addresses do **not** use the draft/publish workflow — every saved address is immediately accessible.

## Fields

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | integer | — | Auto-generated unique identifier |
| `title` | string | Yes | Label for the address (e.g., "Home", "Office") |
| `name` | string | Yes | Full name of the recipient at this address |
| `address` | string | Yes | Street address line |
| `city` | string | Yes | City name |
| `state` | string | Yes | State or province |
| `postalCode` | string | Yes | Postal or ZIP code |
| `phone` | string | Yes | Contact phone number (excluding dial code) |
| `dialCode` | string | No | International dialing code (e.g., `"+34"`) |
| `user` | relation → User | No | The user this address belongs to |

## Relationships

| Relation | Type | Target model | Description |
|---|---|---|---|
| `user` | One-to-one | users-permissions.user | The account that owns this address |

## Notes

<Note>
  **No draft/publish** — The `Address` collection type has `draftAndPublish` disabled. All address records are immediately live once created.
</Note>

<Note>
  **User scoping** — Addresses are linked to a user via the `user` relation. API requests for addresses should always be filtered by the authenticated user's ID to prevent cross-user data exposure. For example: `GET /api/addresses?filters[user][id][$eq]=55`.
</Note>

<Note>
  **Dial code** — The `dialCode` field stores the country calling code separately from the local `phone` number (e.g., `dialCode: "+34"`, `phone: "612 345 678"`). Concatenate them to form the full international number.
</Note>

<Note>
  **Order snapshots** — When an order is placed, address data is copied into the [Order](/models/order) `addressShiping` field. Editing or deleting an `Address` record does not retroactively change existing orders.
</Note>
